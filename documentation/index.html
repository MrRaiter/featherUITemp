Introduction

This documentation is to help you regarding install, set up & customization of the app. Please go through the documentation carefully to understand how it works.

Feather has been Kick-started with "create-react-app". You can find detailed documentation about it at CreateReactApp.

React 17 ready

React lifecycle methods have received an important change lately and we’ve updated all custom components and pages according to the new suggested APIs.

Setup environment #back to top

In order to kick start your project you will need a react development environment, please install the following:

1. Install required software

Node.js

Node.js® is a JavaScript runtime built on Chrome's V8 JavaScript engine. Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient.

Yarn

Yarn is a dependency management software far superior to NPM, we recommend that you use yarn instead of npm.

Ruby

Ruby is being used to write Sass styling. If your OS is Windows download and install Ruby from. If you are using MacOS you already have it, you can skip this step.

Sass lang

If you don't have sass installed, run this command in terminal: "yarn global add sass". You can find out more at https://sass-lang.com/install.

2. Install Feather dependency node modules

Open terminal and navigate to the Feather template folder.
Run: yarn install
This will install all node_modules including React.

3. Start in development mode

Open terminal and navigate to the Feather template folder.
Run: yarn start
The above command will start webpack server, opening your browser, than listen for changes in the code.

4. Build for production

Open terminal and navigate to the Feather template folder.
Run: yarn build
The above command will create an optimized build. All the code will be moved to the build folder. Now you can upload all the content of your build folder to your live web server.

BUILD NOTE!

If you don't plan to upload the app within the root of your domain name ( ex: http://yourdomain.com ), in order for the react router to work properly: Open terminal and navigate to the Feather template folder, open "package.json" file add "homepage" variable within the root of the object (ex: "homepage": "http://yourdomain.com/my_app", ). Don't forget the comma!

Folders & files structure #back to top

SRC folder

Both JS and styles sources can be found under "src" foder See src files structure (PDF)

index.js, main

The app entry point is index.js, which will load ./main/Main.js with Authentication If a user is authenticated Main.js will fall to private routes, if not to public routes (See Menu and routes section).

Components

You can find all component under the ./components folder, since most of the components are "smart" I did not separate them to a "containers" folder also.

Pages

You can find all pages under the ./pages folder. All pages are asynchronous loaded (See Menu and routes section).
We've also built a ./pages/base/GenericPage that should be used by any page you create. The "GenericPage" will handle content scroll, resizing, helpers for page title, page navigation and additional top page navs. Please see examples on different pages we've created.

Styles

You can find all styles under the ./styles folder. You should only edit the styles from the sass folder, when running yarn start or yarn build they will be moved to the css folder than minified and moved to the build/static folder.

PUBLIC folder

"public" folder contains index.html, third party assets and dummy data for fake API.

  1. Edit index.html to change the title of the page or include additional assets.
  2. Edit index.html to add your own Google Maps API Key (otherwise the map page will not work).
  3. Font awesome is also included within public/assets/css, you can use it's icons as you like through out the app.
  4. Dummy data - we've added some dummy JSONs and images that are being used by fake API to load data within the app. Most of the components from Feather are wired to Redux and a Fake API, this way you can only change the API's route to a real one and and have fully functional UIs without touching the code.

NOTE! - some of the responses's output is HTML, this way you can have more flexibility designing the output items without touching the code or have different design for the same component. Example: you might want a different design for search users responses compared to search invoices responses.

Menus & routes#back to top

Routes

You can find all routes at ./routes/Routes.js. We defined routes in a single place because same route can be used within different parts of the app ( Ex: menu, Router ), this way when one would change a route path for example, changes will be applied all over.

Routes can be public or private (see Folders and files structure section).

Public routes In order to make a route public just add isPublic: true. As an example signin and signup are public routes that do not require authentication. The Switch component that will display components with public routes can be found at ./routes/PublicRoutes.js

Private routes The Switch component that will display components with private routes can be found at ./routes/PrivateRoutes.js

The logic that will decide which routes should be displayed can be found within ./main/Main.js and ./components/utils/RequireAuth.js

Menus

All Menu items can be found at ./components/menus/main/MainMenuItems.js. The implementation is pretty straight forward, just use MenuLink and Submenu components to render the menu.
At this moment Feather only supports one level submenu.

Authentication#back to top

JWT token based

Feather promotes a JWT token based authentication. Investigate authenticate action within ./actions/index.js. A JWT token is stored within ./services/StorageService.js, than it's being added to axios as an interceptor within ./services/API. This way all request to the server will carry the token as "x__authorization" header.

As you might prefer you can change LocalStorage to Session storage.

Translation #back to top

You can find out translation sample at ./config/translation.js. Not all the app is translated, you can find samples on how to for all menu items and all pages titles.

In order to create a new language open/edit ./config/constants.js and add a new LANG_OPTIONS ( country_code and label ). You also need to open/edit ./config/translation.js and add a new key to the root object, the key should be named as the country code.

Look & feel #back to top

Feather uses material-ui V1, you can override the theme within ./theme/Customize.js. You can find out more at https://material-ui.com/customization/overrides/

Further customization can be done through styles. Open / edit ./styles/sass/colors.scss, ./styles/sass/settings.scss.
